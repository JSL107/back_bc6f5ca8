# 문제 2 - NEIS 급식 API

## 왜 이 서비스를 만들었는가

학교 급식 데이터는 NEIS에서 조회할 수 있지만, 실제 서비스 화면이나 가정통신문에 바로 쓰기에는 형태가 거칠고 해석 비용이 큽니다.  
저는 이 "조회 가능함"과 "실사용 가능함" 사이의 간격을 줄이는 API가 필요하다고 판단했습니다.

이 서비스를 만든 이유는 아래 3가지입니다.

1.  원본 데이터는 사람이 바로 읽기 어렵습니다.  
    메뉴 문자열에 알레르기 코드가 섞여 있고, 영양 정보도 가공 전 텍스트라서 클라이언트가 매번 파싱 로직을 중복 구현해야 합니다.

2.  학부모 관점의 핵심 정보가 부족합니다.  
    특히 알레르기 보유 자녀 기준으로 "주의가 필요한 메뉴"를 빠르게 확인할 수 있어야 하는데, 원본 응답은 그 목적에 맞지 않습니다.

## 서비스 목표

-   가정통신문/앱에서 바로 쓰는 응답 형태 제공
-   자녀 알레르기 기반 경고 정보 제공
-   외부 API 의존성을 서버에서 흡수해 안정성 확보
-   이후 주간 요약/통계 기능으로 확장 가능한 기반 마련

## 프로젝트 개요

이 프로젝트는 NEIS 급식 데이터를 학부모/가정통신문 관점에서 바로 사용할 수 있는 형태로 가공하는 NestJS 기반 API 서버입니다.

## API 요약

앱 공통 프리픽스/버전: `/api/v1`

| Method | Path | 설명 |
| :----- | :--- | :---------------------------------- |
| GET    | `/api/v1/meals` | 급식 목록 및 평균 영양소 조회 (지정 안 하면 당일 데이터, 알레르기 지정 시 해당 날짜만 필터링) |

## 표방한 구조와 선택 이유

구조는 `Controller -> Service -> Repository -> Client` 계층형으로 잡았습니다.

이 구조를 택한 이유는 "복잡하게 보이더라도 운영/변경 비용이 더 낮기 때문"입니다.

## 핵심 결정 포인트

### 1) raw 데이터 캐시 + 요청별 경고 계산

알레르기(`childAllergies`)는 사용자마다 달라지는 값이라 캐시 키에 포함하면 캐시가 거의 쪼개집니다.  
그래서 캐시는 "학교/기간/식사유형" 기준으로만 두고, 경고 여부만 요청 시점에 계산했습니다.

### 2) 입력 기본값을 API 경계에서 정규화

`fromDate`, `toDate`가 없을 때 "이번 주 월~금"을 기본으로 채우는 처리를 DTO 변환 단계에서 수행했습니다.  
Service는 항상 정규화된 값을 받으므로 핵심 로직에만 집중할 수 있습니다.

### 3) 예외 포맷 통일

Global Exception Filter를 둔 이유는 장애 원인 자체보다 "클라이언트가 일관된 포맷으로 해석 가능"한 것을 우선했기 때문입니다.  
검증 실패/런타임 오류 모두 `{ statusCode, error }` 형식으로 반환합니다.

## 구현 결과

-   식단 및 영양정보 조회: `GET /api/v1/meals`
    - `fromDate`, `toDate`를 생략하면 **오늘 하루** 데이터가 반환됩니다.
    - `childAllergies` 파라미터를 제공하면, 해당 알레르기 유발 물질이 포함되어 **WARNING이 추가된 날짜만 필터링**되어 반환됩니다.
    ```bash
    curl "http://localhost:3000/api/v1/meals?officeCode=B10&schoolCode=7010536&fromDate=20240415&toDate=20240419&mealType=2&childAllergies=1,5"
    ```
    ```json

        "date": "20240415",
        "mealType": 2,
        "menu": [
          { "name": "현미밥", "allergies": [] },
          { "name": "쇠고기미역국", "allergies": ["16"] },
          { "name": "오징어볶음", "allergies": ["5", "6", "17"] },
          { "name": "배추김치", "allergies": ["9"] },
    ```

-   전역 검증/예외 처리, 외부 API 타임아웃, in-memory 캐시(TTL 10분) 적용
-   메뉴 파싱/알레르기 매핑/영양소 파싱/요약 계산 로직 단위 테스트 작성
